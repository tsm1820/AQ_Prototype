# Original Requirement - 21 Apr 2023
---
จะทำเว็บแอพ ที่เป็นระบบทำ Adaptive Quiz ขึ้นโครงโค้ดบางส่วนแล้ว ทำเป็น GUI ผ่าน tkinter แต่ยังไม่ได้ convert ทั้งหมดให้เป็น flask.

ต้องการขึ้นโครงทั้งหมดเป็น demo สำหรับ uat

 สร้างเว็บแอป Adaptive Quiz ใน flask framework พร้อมใช้งานในการทดสอบ demo/prototype ใน UAT โดยมีการแปลงไฟล์แสดงผล GUI ใน tkinter ไปเป็นเว็บแอป (routing api)
	• ไฟล์มีตัว adaptive quiz engine ทำหน้าที่เลือกข้อคำถามจาก learner ability
	• ไฟล์ question mapping ทำหน้าที่เหมือน quiz pool และคอยทำหน้าที่แสดงคำถาม
• ส่วน User interface ประกอบไปด้วยหน้า homepage/ configuration page/ quiz page/ feedback result page โดยหน้าแรกจะให้กรอก Learner ID และมีปุ่ม start และ exit ส่วนหน้าตั้งค่าจะมีให้เลือกเวลาทำข้อสอบ จำนวนข้อคำถาม และมีตัวเลือกให้เลือกเซลล์

• หน้าแสดงคำถามจะมีคำถาม ตัวเลือก ปุ่ม submit และปุ่ม I don't know พร้อมคำอธิบายเมื่อตอบคำถามถูกหรือผิด ส่วนหน้าสรุปผลจะแสดงคะแนนรวม

• pop up ถามเสมอว่าจะยืนยันการตอบนี้ อยากให้ progress tracking ขึ้นว่าไปถึงข้อที่เท่าไรแล้วด้วยครับ มี timer โชว์ด้วย เมื่อตอบแล้วจะบอกว่าถูกหรือผิด พร้อม explanation (หากเขียนโค้ดเว้นไว้ให้รับข้อมูลจากไฟล์เดี๋ยวผมไปเติม explanation อีกทีครับ)
• ตรง IRT parameters Data ตอนนี้รับค่าจากไฟล์ excel (traindata.xlsx) อยากให้มันฝังไปDB เลยไม่ต้องอัพโหลดทุกครั้ง ไม่แน่ใจต้องสร้าง DB SQL เพิ่มด้วยไหมครับแบบนี้

Quiz page
  - Hide / show explanation สามารถทำได้ครับ แต่ว่า ข้อมูล explanation จะให้แสดงเมื่อไหร่ครับ และ Hide/show สามารถกดได้ทันทีเมื่ออยู่ใน quiz โดยที่ยังไม่ submit answer หรือไม่ครับ

  - โชว์คำตอบว่า Correct หรือ Incorrect ข้างล่าง ทำปุ่ม Hide/show explanation ได้ไหมครับ เพิ่มเติมอยากได้สีเขียวแทน Correct และ แดง แทน Incorrect ครับ, Submit กับ Next อยู่ใกล้กันแทนได้ไหมครับ

    - ปุ่ม Next ให้แสดงตลอดเวลา ใช่ไหมครับ แล้วก็กดได้เฉพาะตอนที่ Submit answer แล้วเท่านั้น ถูกไหมครับ

ตรงนี้ขอสอบถามหน่อยครับ
ตอนนี้ถ้าเราเลือก จำนวนข้างล่างเป็น 4 แล้วข้างบนเป็น 10 คำถามทั้งหมดมันจะเป็น 40 (4 x 10)

คือพอจะเป็นไปได้ไหมที่เราจะเลือกข้างล่าง 4 แล้วคอลไปที่ quizpool บอกว่าดึงคำถามที่อยู่ใน cell มากองไว้ในอีกพูล (temporary) ทีนี้ก็จะเลือกคำถามจากพูลนี้แทน ทั้งหมดจำนวน 10 ข้อ (เลือกคำถามยังคงรันตาม rule ที่เซ็ตไว้) อันนี้พอจะแก้จากตัวโมเดลปัจจุบันได้ไหมครับ
  คือจากเดิม ดึงทีละครั้งทีละ จาก cell ทีละ cell เป็น ดึง cell / quiz ทีเดียวแล้วกองหรือครับ
  ตอนแรกก็คิดว่าจะทำอย่างนั้นครับ แต่ว่า ไม่แน่ใจจะมีผลกับตอน calculate probability หรือเปล่าน่ะครับ
  เพราะผมเห็นว่าเขียน function มาค่อนข้างผูกกันอยู่
  เลยใช้ วิธีเดิมไปก่อน แต่ทั้งนี้ทั้งนั้นต้องแจ้งว่า ถ้าหากจะแก้จริง อาจจะต้องรื้อใหม่อีกรอบครับ
    จากเดิมคือดึง cell มาก่อนแล้วใช้แค่ไอเทมจาก cell นั้น ถูกแล้วครับ แต่ถ้าต้องแกะโมเดลใหม่อาจจะใช้เวลานานไหมครับ? 
    ผมพอจะนึกวิธีอีกวิธีนึงแต่ไม่รู้มันจะยากไหม
    ก็คือ เลือก จำนวนเซลล์เหมือนเดิม เลือกไอเทมแบบเดิม แต่สามารถลิมิตมันไว้ที่ 10 ข้อ ( หรือจำนวนอื่นๆ) 
    เพราะตอนนี้ถ้ากดปุ่มนั้นเหมือน user อาจจะเข้าใจผิดได้ครับว่าต้องทำแค่ 10 ข้อ (แต่ความเป็นเยอะกว่ามาก)
      ใช้ประมาณ 2-3 วันครับเพิ่มเติม ลิมิตไว้ที่ 10 ข้ออันนี้หมายถึง จำนวน (quiz x cell) หรือเปล่าครับ
      จริงๆ ตรงส่วน answer history ผมทำเพิ่มให้ได้นะครับ ว่าเป็น กี่ข้อ / ทั้งหมดที่ทำไปแล้ว
        คือตอนนี้ขั้นต่ำมันคือ 3 แน่ๆ (quiz x cell) แต่อยากให้มันลิมิตไว้ที่10ครับ สมมติเลือก 4cells x 3 quizzes =12 แต่ไม่ต้องการให้มันเกินไปถึง12 อะไรประมาณนี่ครับ
